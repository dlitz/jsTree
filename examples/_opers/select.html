Selecting nodes

This is the xml_flat example with the addition of rules for selecting nodes.

The settings.rules.clickable object can either be set to "all", "none" or an array of string values. Each of the items of the array is compared against the settings.rules.type_attr attribute (which defaults to "rel").

In this example nodes of type "folder" and "file" are clickable, "locked" and "root" are not clickable.

* The id or array of ids in the selected setting is/are selected onload.

* The array of ids in the opened setting are opened onload.

* You could also check the metadata examples for inline declaration of clickable.



javascript$(function() { tree1 = new tree_component(); tree1.init($("#demo"), { data : { type : "xml_flat", url : "flat.xml" }, opened : ["#n2"], selected : "#n6", rules : { clickable : [ "folder" , "file" ] } }); }); xml<?xml version="1.0" encoding="UTF-8"?> <root> <item id="n1" parent_id="0" rel="root" state="open"><content><name icon="../images/hd.png">Root node</name></content></item> <item id="n2" parent_id="1" rel="folder"><content><name>Child 1</name></content></item> <item id="n6" parent_id="1" rel="locked"><content><name icon="../images/locked.png">Child 1</name></content></item> <item id="n3" parent_id="1" rel="folder" title="My name is child 2"><content><name>Child 2</name></content></item> <item id="n4" parent_id="2" rel="file"><content><name icon="../images/file.png">A sub child</name></content></item> <item id="n5" parent_id="2" rel="file"><content><name icon="../images/file.png">Sub-children ...</name></content></item> </root>
