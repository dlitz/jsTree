Moving/copying nodes

This is the xml_flat example with the addition of rules for moving nodes.

The settings.rules.draggable object can either be set to "all", "none" or an array of string values. Each of the items of the array is compared against the settings.rules.type_attr attribute (which defaults to "rel").

The settings.rules.dragrules object can either be set to "all", "none" or an array of string values. Each of the items of the array is in the form NODE_TYPE POSITION NODE_TYPE. The wildcard "*" is supported. In this example the user can only put files in folders and reorder them.

In this example the user can only drag nodes of type "file".

* You could also check the metadata examples for inline declaration of the draggable, max_children, valid_children rules.

If you wish to copy the nodes when dragging modify the settings.rules.drag_copy option - if set to "on" a copy will always be performed, if set to "ctrl" a copy will only be performed if the user holds the ctrl key down while dropping the nodes. The default value is false - copy not available - move only.
